Welcome

BrainrotFilter is a network-level YouTube content filter. It analyzes videos using keyword matching, scene detection, and audio analysis, then blocks or warns based on configurable thresholds.

This wizard will configure your Linux system automatically, setting up Squid proxy, SSL interception, and iptables rules. Click Next to begin.

Network Setup

Choose how BrainrotFilter sits on your network. Transparent bridge mode is recommended for physical appliances: both NICs become members of an L2 bridge (br0) and client traffic passes through unchanged. Single-NIC mode is for testing or VMs where you want to intercept traffic on one interface only.

Transparent bridge (2 NICs inline) Appliance
Plug WAN NIC to your uplink and LAN NIC to your switch. Both NICs join br0; the appliance itself gets a management IP on br0. Client traffic passes through transparently, with iptables REDIRECT applied on the bridge.
Single-NIC proxy (testing / VM) Simple
One interface handles everything -- useful for VM labs or when you only need to intercept traffic on a single LAN. No bridge is created.

SSL Pinning / Mobile App Bypass

YouTube's iOS and Android apps use SSL certificate pinning, which bypasses Squid's SSL inspection. The following strategies can help:

None / Skip Default
Skip SSL pinning bypass. SSL interception will work for browsers but YouTube mobile apps may bypass filtering.
Option 1: Block YouTube App Update Domains (DNS) Easiest
Blocks domains used by the YouTube app for certificate validation via /etc/hosts entries.
Option 2: Block YouTube App, Use Browser Instead Strict
Adds iptables rules to block the YouTube app's API endpoints. Users must use a browser where SSL bump works.

Install CA Certificate on Your Devices

For SSL interception to work, each device on your network needs to trust the BrainrotFilter CA certificate.

⬇ Download CA Certificate
CA certificate not available for download. Check /etc/brainrotfilter/ssl/ for the CA files.
iPhone / iPad (iOS)
  1. AirDrop or email the .crt file to the device, or open the download link in Safari.
  2. Tap Allow when prompted to download the profile.
  3. Go to Settings → General → VPN & Device Management.
  4. Tap the BrainrotFilter CA profile and tap Install.
  5. Go to Settings → General → About → Certificate Trust Settings.
  6. Enable Full Trust for the BrainrotFilter CA certificate.
Android
  1. Transfer the .crt file to the device (email, USB, or download via browser).
  2. Go to Settings → Security → Encryption & Credentials.
  3. Tap Install a Certificate → CA Certificate.
  4. Select the downloaded BrainrotFilter-CA.crt file.
  5. Confirm the installation when prompted.
Windows
  1. Download the .crt file and double-click it.
  2. Click Install Certificate.
  3. Select Local Machine and click Next.
  4. Choose Place all certificates in the following store.
  5. Click Browse and select Trusted Root Certification Authorities.
  6. Click Next, then Finish. Confirm the security warning.
macOS
  1. Download the .crt file and double-click it.
  2. Keychain Access will open. Add to the System keychain.
  3. Find the certificate in Keychain Access, double-click it.
  4. Expand Trust and set to Always Trust.
  5. Close the window and enter your password to confirm.
Linux
  1. Copy the CA certificate: sudo cp BrainrotFilter-CA.crt /usr/local/share/ca-certificates/
  2. Update the CA store: sudo update-ca-certificates
  3. For Firefox: go to Preferences → Privacy & Security → Certificates → Import.
Quick test: Open YouTube on a device connected to your network and try watching a video. Videos with brainrot content will be analyzed and blocked based on your thresholds.
